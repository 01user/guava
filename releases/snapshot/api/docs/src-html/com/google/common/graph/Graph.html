017package com.google.common.graph;
018
019import com.google.common.annotations.Beta;
020
021import java.util.Set;
022
023import javax.annotation.Nullable;
024
025/**
026 * A graph consisting of a set of nodes of type N and a set of (implicit) edges.
027 * Users that want edges to be first-class objects should use the subtype {@link Network} instead.
028 *
029 * <p>For convenience, we may use the term 'graph' to refer to any subtype of {@link Graph},
030 * including {@link Network} and {@link Hypergraph}.
031 *
032 * <p>Users that wish to modify a {@code Graph} must work with its subinterface,
033 * {@link MutableNetwork}.
034 *
035 * <p>This interface permits, but does not enforce, any of the following variations of graphs:
036 * <ul>
037 * <li>directed and undirected edges
038 * <li>hyperedges (edges which are incident to arbitrary sets of nodes)
039 * <li>nodes and edges with attributes (for example, weighted edges)
040 * <li>nodes and edges of different types (for example, bipartite or multimodal graphs)
041 * <li>internal representations as matrices, adjacency lists, adjacency maps, etc.
042 * </ul>
043 *
044 * <p>Extensions or implementations of this interface may enforce or disallow any or all
045 * of these variations.
046 *
047 * <p>Definitions:
048 * <ul>
049 * <li>{@code node1} and {@code node2} are mutually <b>adjacent</b> (or <b>connected</b> in
050 *     {@code graph} if an edge has been added between them:
051 *     <br><pre><code>
052 *       graph.addEdge(node1, node2);  // after this returns, node1 and node2 are adjacent
053 *     </pre></code>
054 *   In this example, if {@code graph} is <b>directed</b>, then:
055 *   <ul>
056 *   <li>{@code node1} is a <b>predecessor</b> of {code node2} in {@code graph}
057 *   <li>{@code node2} is a <b>successor</b> of {@code node1} in {@code graph}
058 *   <li>{@code node1} has an (implicit) outgoing edge to {@code node2} in {@code graph}
059 *   <li>{@code node2} has an (implicit) incoming edge from {@code node1} in {@code graph}
060 *   </ul>
061 *   If {@code graph} is <b>undirected</b>, then:
062 *   <ul>
063 *   <li>{@code node1} and {@code node2} are mutually predecessors and successors
064 *       in {@code graph}
065 *   <li>{@code node1} has an (implicit) edge in {@code graph} that is both outgoing to
066 *       to {@code node2} and incoming from {@code node2}, and vice versa.
067 *   </ul>
068 * <li>A self-loop is an edge that connects a node to itself.
069 * </ul>
070 *
071 * <p>General notes:
072 * <ul>
073 * <li><b>Nodes must be useable as {@code Map} keys</b>:
074 *   <ul>
075 *   <li>They must be unique in a graph: nodes {@code node1} and {@code node2} are considered
076 *       different if and only if {@code node1.equals(node2) == false}.
077 *   <li>If graph elements have mutable state:
078 *     <ul>
079 *     <li>the mutable state must not be reflected in the {@code equals/hashCode} methods
080 *         (this is discussed in the {@code Map} documentation in detail)
081 *     <li>don't construct multiple elements that are equal to each other and expect them to be
082 *         interchangeable.  In particular, when adding such elements to a graph, you should
083 *         create them once and store the reference if you will need to refer to those elements
084 *         more than once during creation (rather than passing {@code new MyMutableNode(id)}
085 *         to each {@code add*()} call).
086 *     </ul>
087 *   </ul>
088 *   <br>Generally speaking, your design may be more robust if you use immutable nodes and
089 * store mutable per-element state in a separate data structure (e.g. an element-to-state map).
090 * <li>There are no Node classes built in.  So you can have a {@code Graph<Integer>}
091 *     or a {@code Graph<Author>} or a {@code Graph<Webpage>}.
092 * <li>This framework supports multiple mechanisms for storing the topology of a graph,
093 *      including:
094 *   <ul>
095 *   <li>the Graph implementation stores the topology (for example, by storing a
096 *       {@code Map<N, N>} that maps nodes onto their adjacent nodes); this implies that the nodes
097 *       are just keys, and can be shared among graphs
098 *   <li>the nodes store the topology (for example, by storing a {@code List<E>} of adjacent nodes);
099 *       this (usually) implies that nodes are graph-specific
100 *   <li>a separate data repository (for example, a database) stores the topology
101 *   </ul>
102 * </ul>
103 *
104 * <p>Notes on accessors:
105 * <ul>
106 * <li>Accessors which return collections may return views of the Graph. Modifications to the graph
107 *     which affect a view (e.g. calling {@code addNode(n)} or {@code removeNode(n)} while iterating
108 *     through {@code nodes()}) are not supported and may result in ConcurrentModificationException.
109 * <li>Accessors which return collections will return empty collections if their inputs are valid
110 *     but no elements satisfy the request (for example: {@code adjacentNodes(node)} will return an
111 *     empty collection if {@code node} has no adjacent nodes).
112 * <li>Accessors will throw {@code IllegalArgumentException} if passed an element
113 *     that is not in the graph.
114 * <li>Accessors take Object parameters rather than generic type specifiers to match the pattern
115 *     set by the Java Collections Framework.
116 * </ul>
117 *
118 * <p>Notes for implementors:
119 * <ul>
120 * <li>For accessors that return a {@code Set}, there are several options for the set behavior,
121 *     including:
122 *     <ol>
123 *     <li>Set is an immutable copy (e.g. {@code ImmutableSet}): attempts to modify the set in any
124 *         way will throw an exception, and modifications to the graph will <b>not</b> be reflected
125 *         in the set.
126 *     <li>Set is an unmodifiable view (e.g. {@code Collections.unmodifiableSet()}): attempts to
127 *         modify the set in any way will throw an exception, and modifications to the graph will be
128 *         reflected in the set.
129 *     <li>Set is a mutable copy: it may be modified, but modifications to the graph will <b>not</b>
130 *         be reflected in the set, and vice versa.
131 *     <li>Set is a modifiable view: it may be modified, and modifications to the graph will be
132 *         reflected in the set (but modifications to the set will <b>not</b> be reflected in the
133 *         graph).
134 *     <li>Set exposes the internal data directly: it may be modified, and modifications to the
135 *         graph will be reflected in the set, and vice versa.
136 *     </ol>
137 *     Note that (1) and (2) are generally preferred. (5) is generally a hazardous design choice
138 *     and should be avoided, because keeping the internal data structures consistent can be tricky.
139 * <li>Prefer extending {@link AbstractGraph} over implementing {@link Graph} directly. This will
140 *     ensure consistent {@link #equals(Object)} and {@link #hashCode()} across implementations.
141 * <li>{@code Multimap}s are not sufficient internal data structures for Graph implementations
142 *     that support isolated nodes (nodes that have no incident edges), due to their restriction
143 *     that a key either maps to at least one value, or is not present in the {@code Multimap}.
144 * </ul>
145 *
146 * <p>Examples of use:
147 * <ul>
148 * <li>Is {@code node} in the graph?
149 * <pre><code>
150 *   graph.nodes().contains(node)
151 * </code></pre>
152 * <li>Traversing an undirected graph node-wise:
153 * <pre><code>
154 *   // Visit nodes reachable from {@code node}.
155 *   void depthFirstTraverse(N node) {
156 *     for (N neighbor : graph.adjacentNodes(node)) {
157 *       if (!isVisited(neighbor)) {
158 *         visit(neighbor);
159 *         depthFirstTraverse(neighbor);
160 *       }
161 *     }
162 *   }
163 * </code></pre>
164 * </ul>
165 *
166 * @author Joshua O'Madadhain
167 * @param <N> Node parameter type
168 * @since 20.0
169 */
170@Beta
171public interface Graph<N> {
172  /** Returns all nodes in this graph. */
173  Set<N> nodes();
174
175  //
176  // Graph properties
177  //
178
179  /**
180   * Returns true if the edges in this graph have a direction associated with them.
181   *
182   * <p>A directed edge is an {@linkplain #outEdges(Object) outgoing edge} of its
183   * {@linkplain #source(Object) source}, and an {@linkplain #inEdges(Object) incoming edge} of its
184   * {@linkplain #target(Object) target}. An undirected edge connects its
185   * {@linkplain #incidentNodes(Object) incident nodes} to each other, and is both an
186   * {@linkplain #outEdges(Object) outgoing edge} and {@linkplain #inEdges(Object) incoming edge}
187   * of each incident node.
188   */
189  boolean isDirected();
190
191  /**
192   * Returns true if this graph allows self-loops (edges that connect a node to itself).
193   * Attempting to add a self-loop to a graph that does not allow them will throw an
194   * {@link UnsupportedOperationException}.
195   */
196  boolean allowsSelfLoops();
197
198  //
199  // Element-level accessors
200  //
201
202  /**
203   * Returns the nodes which have an incident edge in common with {@code node} in this graph.
204   *
205   * @throws IllegalArgumentException if {@code node} is not an element of this graph
206   */
207  Set<N> adjacentNodes(Object node);
208
209  /**
210   * Returns all nodes in this graph adjacent to {@code node} which can be reached by traversing
211   * {@code node}'s incoming edges <i>against</i> the direction (if any) of the edge.
212   *
213   * @throws IllegalArgumentException if {@code node} is not an element of this graph
214   */
215  Set<N> predecessors(Object node);
216
217  /**
218   * Returns all nodes in this graph adjacent to {@code node} which can be reached by traversing
219   * {@code node}'s outgoing edges in the direction (if any) of the edge.
220   *
221   * <p>This is <i>not</i> the same as "all nodes reachable from {@code node} by following outgoing
222   * edges" (also known as {@code node}'s transitive closure).
223   *
224   * @throws IllegalArgumentException if {@code node} is not an element of this graph
225   */
226  Set<N> successors(Object node);
227
228  //
229  // Element-level queries
230  //
231
232  /**
233   * Returns the number of edges incident in this graph to {@code node}.  If this node has more than
234   * {@code Integer.MAX_VALUE} incident edges in this graph, returns {@code Integer.MAX_VALUE}.
235   *
236   * <p>Note that self-loops only count once towards a node's degree.
237   * This is consistent with the definition of {@link #incidentEdges(Object)}.
238   *
239   * @throws IllegalArgumentException if {@code node} is not an element of this graph
240   */
241  int degree(Object node);
242
243  /**
244   * Returns the number of incoming edges in this graph of {@code node}.  If this node has more than
245   * {@code Integer.MAX_VALUE} incoming edges in this graph, returns {@code Integer.MAX_VALUE}.
246   *
247   * @throws IllegalArgumentException if {@code node} is not an element of this graph
248   */
249  int inDegree(Object node);
250
251  /**
252   * Returns the number of outgoing edges in this graph of {@code node}.  If this node has more than
253   * {@code Integer.MAX_VALUE} outgoing edges in this graph, returns {@code Integer.MAX_VALUE}.
254   *
255   * @throws IllegalArgumentException if {@code node} is not an element of this graph
256   */
257  int outDegree(Object node);
258
259  /**
260   * Returns {@code true} iff {@code object} is a graph that has the same node relationships
261   * as those in this graph.
262   *
263   * <p>Thus, two graphs A and B are equal if <b>all</b> of the following are true:
264   * <ul>
265   * <li>A and B have the same node set
266   * <li>A and B have the same adjacency relationships, e.g., for each {@code node} the sets of
267   *     successor and predecessor nodes are the same in both graphs.
268   *     <br>This implies that every edge in A and B connects the same nodes in the same
269   *     direction (if any).
270   * </ul>
271   *
272   * <p>Graph properties are <b>not</b> respected by this method. For example, two relations may
273   * be considered equal even if one allows self-loops and the other doesn't. Additionally, the
274   * order in which edges or nodes are added to the graph, and the order in which they are
275   * iterated over, are irrelevant.
276   *
277   * <p>A reference implementation of this is provided by {@link Graphs#equal(Graph, Graph)}.
278   */
279  @Override
280  boolean equals(@Nullable Object object);
281
282  /**
283   * Returns the hash code for this graph. The hash code of a graph is defined as the hash code
284   * of a map from each of the graph's nodes to their adjacent nodes.
285   *
286   * <p>A reference implementation of this is provided by {@link Graphs#hashCode(Graph)}.
287   *
288   * <p>Note that by this definition, two graphs that are equal in every aspect except edge
289   * direction will have the same hash code (but can still be differentiated by {@link #equals}.
290   */
291  @Override
292  int hashCode();
293}
