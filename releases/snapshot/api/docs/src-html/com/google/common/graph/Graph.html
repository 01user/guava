017package com.google.common.graph;
018
019import com.google.common.annotations.Beta;
020import com.google.errorprone.annotations.CanIgnoreReturnValue;
021
022import java.util.Set;
023
024import javax.annotation.Nullable;
025
026/**
027 * A graph consisting of a set of nodes of type N and a set of edges of type E.
028 *
029 * <p>This interface permits, but does not enforce, any of the following variations of graphs:
030 * <ul>
031 * <li>directed and undirected edges
032 * <li>hyperedges (edges which are incident to arbitrary sets of nodes)
033 * <li>nodes and edges with attributes (for example, weighted edges)
034 * <li>nodes and edges of different types (for example, bipartite or multimodal graphs)
035 * <li>parallel edges (multiple edges which connect a single set of vertices)
036 * <li>internal representations as matrices, adjacency lists, adjacency maps, etc.
037 * </ul>
038 *
039 * <p>Extensions or implementations of this interface may enforce or disallow any or all
040 * of these variations.
041 *
042 * <p>Definitions:
043 * <ul>
044 * <li>{@code edge} and {@code node} are <b>incident</b> to each other if the set of
045 *     {@code edge}'s endpoints includes {@code node}.
046 * <li>{@code node1} and {@code node2} are mutually <b>adjacent</b> if both are incident
047 *     to a common {@code edge}.
048 *     <br>Similarly, {@code edge1} and {@code edge2} are mutually adjacent if both are
049 *     incident to a common {@code node}.
050 * <li>Elements are <b>connected</b> if they are either incident or adjacent.
051 * <li>{@code edge} is an <b>incoming edge</b> of a {@code node} if it can be traversed (in
052 *     the direction, if any, of {@code edge}) from a node adjacent to {@code node}.
053 * <li>{@code edge} is an <b>outgoing edge</b> of {@code node} if it can be traversed (in
054 *     the direction, if any, of {@code edge}) from {@code node} to reach a node adjacent to
055 *     {@code node}.
056 *   <ul>
057 *   <li>Note: <b>undirected</b> edges are both incoming and outgoing edges of a {@code node},
058 *       while <b>directed</b> edges are either incoming or outgoing edges of {@code node}
059 *       (and not both, unless the edge is a self-loop).
060 *       <br>Thus, in the following example {@code edge1} is an incoming edge of {@code node2} and
061 *       an outgoing edge of {@code node1}, while {@code edge2} is both an incoming and an outgoing
062 *       edge of both {@code node3} and {@code node4}:
063 *       <br><pre><code>
064 *         directedGraph.addEdge(edge1, node1, node2);
065 *         undirectedGraph.addEdge(edge2, node3, node4);
066 *       </pre></code>
067 *   </ul>
068 * <li>A node {@code pred} is a <b>predecessor</b> of {@code node} if it is incident to an incoming
069 *     {@code edge} of {@code node} (and is not itself {@code node} unless {@code edge} is
070 *     a self-loop).
071 * <li>A node {@code succ} is a <b>successor</b> of {@code node} if it is incident to an outgoing
072 *     {@code edge} of {@code node} (and is not itself {@code node} unless {@code edge} is
073 *     a self-loop).
074 * <li>Directed edges only:
075 *   <ul>
076 *   <li>{@code node} is a <b>source</b> of {@code edge} if {@code edge} is an outgoing edge
077 *       of {@code node}.
078 *   <li>{@code node} is a <b>target</b> of {@code edge} if {@code edge} is an incoming edge
079 *       of {@code node}.
080 *   </ul>
081 * </ul>
082 *
083 * <p>General notes:
084 * <ul>
085 * <li><b>Nodes/edges must be useable as {@code Map} keys</b>:
086 *   <ul>
087 *   <li>They must be unique in a graph: nodes {@code node1} and {@code node2} are considered
088 *       different if and only if {@code node1.equals(node2) == false}, and the same for edges.
089 *   <li>If you would otherwise have duplicate edges (e.g. weighted edges represented by a Double),
090 *       you can instead wrap the edges in a custom class that defers to {@link Object} for its
091 *       {@code equals()} and {@code hashCode()} implementations.
092 *   <li>If graph elements have mutable state, both of the following must be true:
093 *     <ul>
094 *     <li>the mutable state must not be reflected in the {@code equals/hashCode} methods
095 *         (this is discussed in the {@code Map} documentation in detail)
096 *     <li>don't construct multiple elements that are equal to each other and expect them to be
097 *         interchangeable.  In particular, when adding such elements to a graph, you should create
098 *         them once and store the reference if you will need to refer to those elements more than
099 *         once during creation (rather than passing {@code new MyMutableNode(id)} to each
100 *         {@code add*()} call).
101 *     </ul>
102 *   </ul>
103 *   <br>Generally speaking, your design may be more robust if you use immutable nodes/edges and
104 * store mutable per-element state in a separate data structure (e.g. an element-to-state map).
105 * <li>There are no Node or Edge classes built in.  So you can have a {@code Graph<Integer, String>}
106 *     or a {@code Graph<Author,Publication>} or a {@code Graph<Webpage,Link>}.
107 * <li>This framework supports multiple mechanisms for storing the topology of a graph, including:
108 *   <ul>
109 *   <li>the Graph implementation stores the topology (for example, by storing a {@code Map<N, E>}
110 *       that maps nodes onto their incident edges); this implies that the nodes and edges
111 *       are just keys, and can be shared among graphs
112 *   <li>the nodes store the topology (for example, by storing a {@code List<E>} of incident edges);
113 *       this (usually) implies that nodes are graph-specific
114 *   <li>a separate data repository (for example, a database) stores the topology
115 *   </ul>
116 * <li>Users that are not interested in edges as first-class objects have a couple of options:
117 *   <ul>
118 *   <li>pass in arbitrary Objects as edges: {@code addEdge(new Object(), n1, n2)}
119 *   <li>create an implementation of (or subinterface of, or class delegating to) Graph
120 *       that only exposes node-related methods.
121 *   </ul>
122 * </ul>
123 *
124 * <p>Notes on accessors:
125 * <ul>
126 * <li>Accessors which return collections may return views of the Graph. Modifications to the graph
127 *     which affect a view (e.g. calling {@code addNode(n)} or {@code removeNode(n)} while iterating
128 *     through {@code nodes()}) are not supported and may result in ConcurrentModificationException.
129 * <li>Accessors which return collections will return empty collections if their inputs are valid
130 *     but no elements satisfy the request (for example: {@code adjacentNodes(node)} will return an
131 *     empty collection if {@code node} has no adjacent nodes).
132 * <li>Accessors will throw {@code IllegalArgumentException} if passed a node/edge
133 *     that is not in the graph.
134 * <li>Accessors take Object parameters rather than N/E generic type specifiers to match the pattern
135 *     set by the Java Collections Framework.
136 * </ul>
137 *
138 * <p>Notes for implementors:
139 * <ul>
140 * <li>For accessors that return a {@code Set}, there are several options for the set behavior,
141 *     including:
142 *     <ol>
143 *     <li>Set is an immutable copy (e.g. {@code ImmutableSet}): attempts to modify the set in any
144 *         way will throw an exception, and modifications to the graph will <b>not</b> be reflected
145 *         in the set.
146 *     <li>Set is an unmodifiable view (e.g. {@code Collections.unmodifiableSet()}): attempts to
147 *         modify the set in any way will throw an exception, and modifications to the graph will be
148 *         reflected in the set.
149 *     <li>Set is a mutable copy: it may be modified, but modifications to the graph will <b>not</b>
150 *         be reflected in the set, and vice versa.
151 *     <li>Set is a modifiable view: it may be modified, and modifications to the graph will be
152 *         reflected in the set (but modifications to the set will <b>not</b> be reflected in the
153 *         graph).
154 *     <li>Set exposes the internal data directly: it may be modified, and modifications to the
155 *         graph will be reflected in the set, and vice versa.
156 *     </ol>
157 *     Note that (1) and (2) are generally preferred. (5) is generally a hazardous design choice
158 *     and should be avoided, because keeping the internal data structures consistent can be tricky.
159 * <li>Prefer extending {@link AbstractGraph} over implementing {@link Graph} directly. This will
160 *     ensure consistent {@link #equals(Object)} and {@link #hashCode()} across implementations.
161 * <li>{@code Multimap}s are not sufficient internal data structures for Graph implementations
162 *     that support isolated nodes (nodes that have no incident edges), due to their restriction
163 *     that a key either maps to at least one value, or is not present in the {@code Multimap}.
164 * </ul>
165 *
166 * <p>Examples of use:
167 * <ul>
168 * <li>Is {@code node} in the graph?
169 * <pre><code>
170 *   graph.nodes().contains(node)
171 * </code></pre>
172 * <li>Traversing an undirected graph node-wise:
173 * <pre><code>
174 *   // Visit nodes reachable from {@code node}.
175 *   void depthFirstTraverse(N node) {
176 *     for (N neighbor : graph.adjacentNodes(node)) {
177 *       if (!isVisited(neighbor)) {
178 *         visit(neighbor);
179 *         depthFirstTraverse(neighbor);
180 *       }
181 *     }
182 *   }
183 * </code></pre>
184 * <li>Traversing a directed graph edge-wise:
185 * <pre><code>
186 *   // Update the shortest-path distances of the successors to {@code node}
187 *   // in a directed graph (inner loop of Dijkstra's algorithm):
188 *   void updateDistances(N node) {
189 *     nodeDistance = distances.get(node);
190 *     for (E outEdge : graph.outEdges(node)) {
191 *       N target = graph.target(outEdge);
192 *       double targetDistance = nodeDistance + outEdge.getWeight();
193 *       if (targetDistance < distances.get(target)) {
194 *         distances.put(target, targetDistance);
195 *       }
196 *     }
197 *   }
198 * </code></pre>
199 * </ul>
200 *
201 * @author Joshua O'Madadhain
202 * @param <N> Node parameter type
203 * @param <E> Edge parameter type
204 * @since 20.0
205 */
206@Beta
207public interface Graph<N, E> {
208  /** Returns all nodes in this graph. */
209  Set<N> nodes();
210
211  /** Returns all edges in this graph. */
212  Set<E> edges();
213
214  //
215  // Graph properties
216  //
217
218  /**
219   * Returns true if the edges in this graph have a direction associated with them.
220   *
221   * <p>A directed edge is an {@linkplain #outEdges(Object) outgoing edge} of its
222   * {@linkplain #source(Object) source}, and an {@linkplain #inEdges(Object) incoming edge} of its
223   * {@linkplain #target(Object) target}. An undirected edge connects its
224   * {@linkplain #incidentNodes(Object) incident nodes} to each other, and is both an
225   * {@linkplain #outEdges(Object) outgoing edge} and {@linkplain #inEdges(Object) incoming edge}
226   * of each incident node.
227   */
228  boolean isDirected();
229
230  /**
231   * Returns true if this graph allows parallel edges. Attempting to add a parallel edge to a graph
232   * that does not allow them will throw an {@link UnsupportedOperationException}.
233   */
234  boolean allowsParallelEdges();
235
236  /**
237   * Returns true if this graph allows self-loops (edges that connect a node to itself).
238   * Attempting to add a self-loop to a graph that does not allow them will throw an
239   * {@link UnsupportedOperationException}.
240   */
241  boolean allowsSelfLoops();
242
243  //
244  // Element-level accessors
245  //
246
247  /**
248   * Returns the edges whose endpoints in this graph include {@code node}.
249   *
250   * @throws IllegalArgumentException if {@code node} is not an element of this graph
251   */
252  Set<E> incidentEdges(Object node);
253
254  /**
255   * Returns the nodes which are the endpoints of {@code edge} in this graph.
256   *
257   * <p>For self-loop edges, the returned set's size will be 1. If the graph is
258   * {@linkplain #isDirected() directed} and {@code edge} is not a self-loop, the
259   * iteration order will be {@code [source(edge), target(edge)]}.
260   *
261   * @throws IllegalArgumentException if {@code edge} is not an element of this graph
262   */
263  Set<N> incidentNodes(Object edge);
264
265  /**
266   * Returns the nodes which have an {@linkplain #incidentEdges(Object) incident edge}
267   * in common with {@code node} in this graph.
268   *
269   * @throws IllegalArgumentException if {@code node} is not an element of this graph
270   */
271  Set<N> adjacentNodes(Object node);
272
273  /**
274   * Returns the edges which have an {@linkplain #incidentNodes(Object) incident node}
275   * in common with {@code edge} in this graph.
276   *
277   * <p>Whether an edge is considered adjacent to itself is not defined by this interface, but
278   * generally for non-hypergraphs, edges are not considered to be self-adjacent.
279   *
280   * @throws IllegalArgumentException if {@code edge} is not an element of this graph
281   */
282  Set<E> adjacentEdges(Object edge);
283
284  /**
285   * Returns the set of edges that connect {@code node1} to {@code node2}.
286   *
287   * <p>This set is the intersection of {@code outEdges(node1)} and {@code inEdges(node2)}. If
288   * {@code node1} is equal to {@code node2}, then it is the set of self-loop edges for that node.
289   *
290   * @throws IllegalArgumentException if {@code node1} or {@code node2} is not an element
291   *     of this graph
292   */
293  Set<E> edgesConnecting(Object node1, Object node2);
294
295  /**
296   * Returns all edges in this graph which can be traversed in the direction (if any) of the edge
297   * to end at {@code node}.
298   *
299   * @throws IllegalArgumentException if {@code node} is not an element of this graph
300   */
301  Set<E> inEdges(Object node);
302
303  /**
304   * Returns all edges in this graph which can be traversed in the direction (if any) of the edge
305   * starting from {@code node}.
306   *
307   * @throws IllegalArgumentException if {@code node} is not an element of this graph
308   */
309  Set<E> outEdges(Object node);
310
311  /**
312   * Returns all nodes in this graph adjacent to {@code node} which can be reached by traversing
313   * {@code node}'s {@linkplain #inEdges(Object) incoming edges} <i>against</i> the direction
314   * (if any) of the edge.
315   *
316   * @throws IllegalArgumentException if {@code node} is not an element of this graph
317   */
318  Set<N> predecessors(Object node);
319
320  /**
321   * Returns all nodes in this graph adjacent to {@code node} which can be reached by traversing
322   * {@code node}'s {@linkplain #outEdges(Object) outgoing edges} in the direction (if any) of the
323   * edge.
324   *
325   * <p>This is <i>not</i> the same as "all nodes reachable from {@code node} by following outgoing
326   * edges" (also known as {@code node}'s transitive closure).
327   *
328   * @throws IllegalArgumentException if {@code node} is not an element of this graph
329   */
330  Set<N> successors(Object node);
331
332  //
333  // Element-level queries
334  //
335
336  /**
337   * For a directed graph, returns the node for which {@code edge} is an outgoing edge.
338   * For an undirected graph, throws an {@link UnsupportedOperationException}.
339   */
340  N source(Object edge);
341
342  /**
343   * For a directed graph, returns the node for which {@code edge} is an incoming edge.
344   * For an undirected graph, throws an {@link UnsupportedOperationException}.
345   */
346  N target(Object edge);
347
348  /**
349   * Returns the number of edges {@linkplain #incidentEdges(Object) incident} in this graph
350   * to {@code node}.  If this node has more than {@code Integer.MAX_VALUE} incident edges
351   * in this graph, returns {@code Integer.MAX_VALUE}.
352   *
353   * <p>Equivalent to {@code incidentEdges(node).size()}.
354   *
355   * <p>Note that self-loops only count once towards a node's degree.
356   * This is consistent with the definition of {@link #incidentEdges(Object)}.
357   *
358   * @throws IllegalArgumentException if {@code node} is not an element of this graph
359   */
360  int degree(Object node);
361
362  /**
363   * Returns the number of {@linkplain #inEdges(Object) incoming edges} in this graph
364   * of {@code node}.  If this node has more than {@code Integer.MAX_VALUE} incoming edges
365   * in this graph, returns {@code Integer.MAX_VALUE}.
366   *
367   * <p>Equivalent to {@code inEdges(node).size()}.
368   *
369   * @throws IllegalArgumentException if {@code node} is not an element of this graph
370   */
371  int inDegree(Object node);
372
373  /**
374   * Returns the number of {@linkplain #outEdges(Object) outgoing edges} in this graph
375   * of {@code node}.  If this node has more than {@code Integer.MAX_VALUE} outgoing edges
376   * in this graph, returns {@code Integer.MAX_VALUE}.
377   *
378   * <p>Equivalent to {@code outEdges(node).size()}.
379   *
380   * @throws IllegalArgumentException if {@code node} is not an element of this graph
381   */
382  int outDegree(Object node);
383
384  //
385  // Element mutations
386  //
387
388  /**
389   * Adds {@code node} to this graph (optional operation).
390   *
391   * <p><b>Nodes must be unique</b>, just as {@code Map} keys must be; they must also be non-null.
392   *
393   * @return {@code true} iff the graph was modified as a result of this call
394   * @throws UnsupportedOperationException if the add operation is not supported by this graph
395   */
396  @CanIgnoreReturnValue
397  boolean addNode(N node);
398
399  /**
400   * Adds {@code edge} to this graph, connecting {@code node1} to {@code node2}
401   * (optional operation).
402   *
403   * <p><b>Edges must be unique</b>, just as {@code Map} keys must be; they must also be non-null.
404   *
405   * <p>If {@code edge} already connects {@code node1} to {@code node2} in this graph
406   * (in the specified order if order is significant, as for directed graphs, else in any order),
407   * then this method will have no effect and will return {@code false}.
408   *
409   * <p>Behavior if {@code node1} and {@code node2} are not already elements of the graph is
410   * unspecified. Suggested behaviors include (a) silently adding {@code node1} and {@code node2}
411   * to the graph or (b) throwing {@code IllegalArgumentException}.
412   *
413   * @return {@code true} iff the graph was modified as a result of this call
414   * @throws IllegalArgumentException if {@code edge} already exists and connects nodes other than
415   *     {@code node1} and {@code node2}, or if the graph is not a multigraph and {@code node1} is
416   *     already connected to {@code node2}
417   * @throws UnsupportedOperationException if the add operation is not supported by this graph
418   */
419  @CanIgnoreReturnValue
420  boolean addEdge(E edge, N node1, N node2);
421
422  /**
423   * Removes {@code node} from this graph, if it is present (optional operation).
424   * In general, all edges incident to {@code node} in this graph will also be removed.
425   * (This is not true for hyperedges.)
426   *
427   * @return {@code true} iff the graph was modified as a result of this call
428   * @throws UnsupportedOperationException if the remove operation is not supported by this graph
429   */
430  @CanIgnoreReturnValue
431  boolean removeNode(Object node);
432
433  /**
434   * Removes {@code edge} from this graph, if it is present (optional operation).
435   * In general, nodes incident to {@code edge} are unaffected (although implementations may choose
436   * to disallow certain configurations, e.g., isolated nodes).
437   *
438   * @return {@code true} iff the graph was modified as a result of this call
439   * @throws UnsupportedOperationException if the remove operation is not supported by this graph
440   */
441  @CanIgnoreReturnValue
442  boolean removeEdge(Object edge);
443
444  /**
445   * Returns {@code true} iff {@code object} is a graph that has the same node/edge relationships
446   * as those in this graph.
447   *
448   * <p>Thus, two graphs A and B are equal if <b>all</b> of the following are true:
449   * <ul>
450   * <li>A and B have the same node set
451   * <li>A and B have the same edge set
452   * <li>A and B have the same incidence relationships, e.g., for each node/edge in A and in B
453   *     its incident edge/node set in A is the same as its incident edge/node set in B.
454   *     <br>Thus, every edge in A and B connect the same nodes in the same direction (if any).
455   * </ul>
456   *
457   * <p>Graph properties are <b>not</b> respected by this method. For example, two graphs may be
458   * considered equal even if one allows parallel edges and the other doesn't. Additionally, the
459   * order in which edges or nodes are added to the graph, and the order in which they are iterated
460   * over, are irrelevant.
461   *
462   * <p>A reference implementation of this is provided by {@link Graphs#equal(Graph, Graph)}.
463   */
464  @Override
465  boolean equals(@Nullable Object object);
466
467  /**
468   * Returns the hash code for this graph. The hash code of a graph is defined as the hash code
469   * of a map from each of the graph's nodes to their incident edges.
470   *
471   * <p>A reference implementation of this is provided by {@link Graphs#hashCode(Graph)}.
472   *
473   * <p>Note that by this definition, two graphs that are equal in every aspect except edge
474   * direction will have the same hash code (but can still be differentiated by {@link #equals}.
475   */
476  @Override
477  int hashCode();
478}
