017package com.google.common.graph;
018
019import com.google.common.annotations.Beta;
020
021import java.util.Set;
022
023import javax.annotation.Nullable;
024
025/**
026 * A network consisting of a set of nodes of type N and a set of edges of type E.
027 * That is, a subtype of {@link Graph} that represents edges as explicit first-class objects.
028 * Users that are not interested in edges as first-class objects should use a {@link Graph}
029 * instead.
030 *
031 * <p>Users that wish to modify a {@code Network} must work with its subinterface,
032 * {@code MutableNetwork}.
033 *
034 * <p>This interface permits, but does not enforce, any of the following variations of graphs:
035 * <ul>
036 * <li>directed and undirected edges
037 * <li>nodes and edges with attributes (for example, weighted edges)
038 * <li>nodes and edges of different types (for example, bipartite or multimodal graphs)
039 * <li>parallel edges (multiple edges which connect a single set of nodes)
040 * </ul>
041 *
042 * <p>Extensions or implementations of this interface may enforce or disallow any or all
043 * of these variations.
044 *
045 * <p>Definitions:
046 * <ul>
047 * <li>{@code edge} and {@code node} are <b>incident</b> to each other if the set of
048 *     {@code edge}'s endpoints includes {@code node}.
049 * <li>{@code nodeA} and {@code nodeB} are mutually <b>adjacent</b> if both are incident
050 *     to a common {@code edge}.
051 *     <br>Similarly, {@code edge1} and {@code edge2} are mutually adjacent if both are
052 *     incident to a common {@code node}.
053 * <li>Elements are <b>connected</b> if they are either incident or adjacent.
054 * <li>{@code edge} is an <b>incoming edge</b> of a {@code node} if it can be traversed (in
055 *     the direction, if any, of {@code edge}) from a node adjacent to {@code node}.
056 * <li>{@code edge} is an <b>outgoing edge</b> of {@code node} if it can be traversed (in
057 *     the direction, if any, of {@code edge}) from {@code node} to reach a node adjacent to
058 *     {@code node}.
059 *   <ul>
060 *   <li>Note: <b>undirected</b> edges are both incoming and outgoing edges of a {@code node},
061 *       while <b>directed</b> edges are either incoming or outgoing edges of {@code node}
062 *       (and not both, unless the edge is a self-loop).
063 *       <br>Thus, in the following example {@code edge1} is an incoming edge of {@code nodeB} and
064 *       an outgoing edge of {@code nodeA}, while {@code edge2} is both an incoming and an outgoing
065 *       edge of both {@code node3} and {@code node4}:
066 *       <br><pre><code>
067 *         directedGraph.addEdge(edge1, nodeA, nodeB);
068 *         undirectedGraph.addEdge(edge2, node3, node4);
069 *       </pre></code>
070 *   </ul>
071 * <li>A node {@code pred} is a <b>predecessor</b> of {@code node} if it is incident to an incoming
072 *     {@code edge} of {@code node} (and is not itself {@code node} unless {@code edge} is
073 *     a self-loop).
074 * <li>A node {@code succ} is a <b>successor</b> of {@code node} if it is incident to an outgoing
075 *     {@code edge} of {@code node} (and is not itself {@code node} unless {@code edge} is
076 *     a self-loop).
077 * <li>Directed edges only:
078 *   <ul>
079 *   <li>{@code node} is a <b>source</b> of {@code edge} if {@code edge} is an outgoing edge
080 *       of {@code node}.
081 *   <li>{@code node} is a <b>target</b> of {@code edge} if {@code edge} is an incoming edge
082 *       of {@code node}.
083 *   </ul>
084 * </ul>
085 *
086 * <p>General notes:
087 * <ul>
088 * <li><b>Nodes/edges must be useable as {@code Map} keys</b>:
089 *   <ul>
090 *   <li>They must be unique in a graph: nodes {@code nodeA} and {@code nodeB} are considered
091 *       different if and only if {@code nodeA.equals(nodeB) == false}, and the same for edges.
092 *   <li>If you would otherwise have duplicate edges (e.g. weighted edges represented by a Double),
093 *       you can instead wrap the edges in a custom class that defers to {@link Object} for its
094 *       {@code equals()} and {@code hashCode()} implementations.
095 *   <li>If graph elements have mutable state:
096 *     <ul>
097 *     <li>the mutable state must not be reflected in the {@code equals/hashCode} methods
098 *         (this is discussed in the {@code Map} documentation in detail)
099 *     <li>don't construct multiple elements that are equal to each other and expect them to be
100 *         interchangeable.  In particular, when adding such elements to a graph, you should create
101 *         them once and store the reference if you will need to refer to those elements more than
102 *         once during creation (rather than passing {@code new MyMutableNode(id)} to each
103 *         {@code add*()} call).
104 *     </ul>
105 *   </ul>
106 *   <br>Generally speaking, your design may be more robust if you use immutable nodes/edges and
107 * store mutable per-element state in a separate data structure (e.g. an element-to-state map).
108 * <li>There are no Node or Edge classes built in.  So you can have a {@code Graph<Integer, String>}
109 *     or a {@code Graph<Author,Publication>} or a {@code Graph<Webpage,Link>}.
110 * </ul>
111 *
112 * <p>Notes on accessors:
113 * <ul>
114 * <li>Accessors which return collections may return views of the Graph. Modifications to the graph
115 *     which affect a view (e.g. calling {@code addNode(n)} or {@code removeNode(n)} while iterating
116 *     through {@code nodes()}) are not supported and may result in ConcurrentModificationException.
117 * <li>Accessors which return collections will return empty collections if their inputs are valid
118 *     but no elements satisfy the request (for example: {@code adjacentNodes(node)} will return an
119 *     empty collection if {@code node} has no adjacent nodes).
120 * <li>Accessors will throw {@code IllegalArgumentException} if passed a node/edge
121 *     that is not in the graph.
122 * <li>Accessors take Object parameters rather than N/E generic type specifiers to match the pattern
123 *     set by the Java Collections Framework.
124 * </ul>
125 *
126 * <p>Notes for implementors:
127 * <ul>
128 * <li>Implementations have numerous options for internal representations: matrices, adjacency
129 *     lists, adjacency maps, etc.
130 * <li>For accessors that return a {@code Set}, there are several options for the set behavior,
131 *     including:
132 *     <ol>
133 *     <li>Set is an immutable copy (e.g. {@code ImmutableSet}): attempts to modify the set in any
134 *         way will throw an exception, and modifications to the graph will <b>not</b> be reflected
135 *         in the set.
136 *     <li>Set is an unmodifiable view (e.g. {@code Collections.unmodifiableSet()}): attempts to
137 *         modify the set in any way will throw an exception, and modifications to the graph will be
138 *         reflected in the set.
139 *     <li>Set is a mutable copy: it may be modified, but modifications to the graph will <b>not</b>
140 *         be reflected in the set, and vice versa.
141 *     <li>Set is a modifiable view: it may be modified, and modifications to the graph will be
142 *         reflected in the set (but modifications to the set will <b>not</b> be reflected in the
143 *         graph).
144 *     <li>Set exposes the internal data directly: it may be modified, and modifications to the
145 *         graph will be reflected in the set, and vice versa.
146 *     </ol>
147 *     Note that (1) and (2) are generally preferred. (5) is generally a hazardous design choice
148 *     and should be avoided, because keeping the internal data structures consistent can be tricky.
149 * <li>Prefer extending {@link AbstractGraph} over implementing {@link Graph} directly. This will
150 *     ensure that the implementations of {@link #equals(Object)} and
151 *     {@link #hashCode()} are mutually consistent, and consistent across
152 *     implementations.
153 * <li>{@code Multimap}s are not sufficient internal data structures for Graph implementations
154 *     that support isolated nodes (nodes that have no incident edges), due to their restriction
155 *     that a key either maps to at least one value, or is not present in the {@code Multimap}.
156 * </ul>
157 *
158 * <p>Examples of use:
159 * <ul>
160 * <li>Is {@code node} in the graph?
161 * <pre><code>
162 *   graph.nodes().contains(node)
163 * </code></pre>
164 * <li>Traversing an undirected graph node-wise:
165 * <pre><code>
166 *   // Visit nodes reachable from {@code node}.
167 *   void depthFirstTraverse(N node) {
168 *     if (!isVisited(node)) {
169 *       visit(node);
170 *       for (N successor : graph.successors(node)) {
171 *         depthFirstTraverse(successor);
172 *       }
173 *     }
174 *   }
175 * </code></pre>
176 * <li>Traversing a directed graph edge-wise:
177 * <pre><code>
178 *   // Update the shortest-path distances of the successors to {@code node}
179 *   // in a directed graph (inner loop of Dijkstra's algorithm):
180 *   void updateDistances(N node) {
181 *     nodeDistance = distances.get(node);
182 *     for (E outEdge : graph.outEdges(node)) {
183 *       N target = graph.target(outEdge);
184 *       double targetDistance = nodeDistance + outEdge.getWeight();
185 *       if (targetDistance < distances.get(target)) {
186 *         distances.put(target, targetDistance);
187 *       }
188 *     }
189 *   }
190 * </code></pre>
191 * </ul>
192 *
193 * @author James Sexton
194 * @author Joshua O'Madadhain
195 * @param <N> Node parameter type
196 * @param <E> Edge parameter type
197 * @since 20.0
198 */
199@Beta
200public interface Network<N, E> extends Graph<N> {
201  /**
202   * Returns all edges in this network, in the order specified by {@link #edgeOrder()}.
203   */
204  Set<E> edges();
205
206  //
207  // Graph properties
208  //
209
210  /**
211   * {@inheritDoc}
212   *
213   * <p>A directed edge is an {@linkplain #outEdges(Object) outgoing edge} of its
214   * {@linkplain Endpoints#source() source}, and an {@linkplain #inEdges(Object) incoming edge}
215   * of its {@linkplain Endpoints#target() target}. An undirected edge connects its
216   * {@linkplain #incidentNodes(Object) incident nodes} to each other, and is both an
217   * {@linkplain #outEdges(Object) outgoing edge} and {@linkplain #inEdges(Object) incoming edge}
218   * of each incident node.
219   */
220  @Override
221  boolean isDirected();
222
223  /**
224   * Returns true if this graph allows parallel edges. Attempting to add a parallel edge to a graph
225   * that does not allow them will throw an {@link UnsupportedOperationException}.
226   */
227  boolean allowsParallelEdges();
228
229  /**
230   * Returns the order of iteration for the elements of {@link #edges()}.
231   */
232  ElementOrder<? super E> edgeOrder();
233
234  //
235  // Element-level accessors
236  //
237
238  /**
239   * Returns the edges whose endpoints in this graph include {@code node}.
240   *
241   * @throws IllegalArgumentException if {@code node} is not an element of this graph
242   */
243  Set<E> incidentEdges(Object node);
244
245  /**
246   * Returns the nodes which are the endpoints of {@code edge} in this graph as {@link Endpoints}.
247   *
248   * @throws IllegalArgumentException if {@code edge} is not an element of this graph
249   */
250  Endpoints<N> incidentNodes(Object edge);
251
252  /**
253   * Returns the edges which have an {@linkplain #incidentNodes(Object) incident node}
254   * in common with {@code edge} in this graph.
255   *
256   * <p>Whether an edge is considered adjacent to itself is not defined by this interface, but
257   * generally edges are not considered to be self-adjacent.
258   *
259   * @throws IllegalArgumentException if {@code edge} is not an element of this graph
260   */
261  Set<E> adjacentEdges(Object edge);
262
263  /**
264   * Returns the set of edges that connect {@code nodeA} to {@code nodeB}.
265   *
266   * <p>This set is the intersection of {@code outEdges(nodeA)} and {@code inEdges(nodeB)}. If
267   * {@code nodeA} is equal to {@code nodeB}, then it is the set of self-loop edges for that node.
268   *
269   * @throws IllegalArgumentException if {@code nodeA} or {@code nodeB} is not an element
270   *     of this graph
271   */
272  Set<E> edgesConnecting(Object nodeA, Object nodeB);
273
274  /**
275   * Returns all edges in this graph which can be traversed in the direction (if any) of the edge
276   * to end at {@code node}.
277   *
278   * @throws IllegalArgumentException if {@code node} is not an element of this graph
279   */
280  Set<E> inEdges(Object node);
281
282  /**
283   * Returns all edges in this graph which can be traversed in the direction (if any) of the edge
284   * starting from {@code node}.
285   *
286   * @throws IllegalArgumentException if {@code node} is not an element of this graph
287   */
288  Set<E> outEdges(Object node);
289
290  //
291  // Element-level queries
292  //
293
294  /**
295   * {@inheritDoc}
296   *
297   * <p>Equivalent to {@code incidentEdges(node).size()}.
298   */
299  @Override
300  int degree(Object node);
301
302  /**
303   * {@inheritDoc}
304   *
305   * <p>Equivalent to {@code inEdges(node).size()}.
306   */
307  @Override
308  int inDegree(Object node);
309
310  /**
311   * {@inheritDoc}
312   *
313   * <p>Equivalent to {@code outEdges(node).size()}.
314   */
315  @Override
316  int outDegree(Object node);
317
318  /**
319   * Returns {@code true} iff {@code object} is a graph that has the same node/edge relationships
320   * as those in this graph.
321   *
322   * <p>Thus, two graphs A and B are equal if <b>all</b> of the following are true:
323   * <ul>
324   * <li>A and B have the same node set
325   * <li>A and B have the same edge set
326   * <li>A and B have the same incidence relationships, e.g., for each node/edge in A and in B
327   *     its incident edge/node set in A is the same as its incident edge/node set in B.
328   *     <br>Thus, every edge in A and B connect the same nodes in the same direction (if any).
329   * </ul>
330   *
331   * <p>Graph properties are <b>not</b> respected by this method. For example, two graphs may be
332   * considered equal even if one allows parallel edges and the other doesn't. Additionally, the
333   * order in which edges or nodes are added to the graph, and the order in which they are iterated
334   * over, are irrelevant.
335   *
336   * <p>A reference implementation of this is provided by
337   * {@link Graphs#equal(Network, Network)}.
338   */
339  @Override
340  boolean equals(@Nullable Object object);
341
342  /**
343   * Returns the hash code for this graph. The hash code of a graph is defined as the hash code
344   * of a map from each of the graph's nodes to their incident edges.
345   *
346   * <p>A reference implementation of this is provided by {@link Graphs#hashCode(Graph)}.
347   *
348   * <p>Note that by this definition, two graphs that are equal in every aspect except edge
349   * direction will have the same hash code (but can still be differentiated by
350   * {@link #equals(Object)}).
351   */
352  @Override
353  int hashCode();
354}
